# DublinISL Controls — Raspberry Pi IP v3

PTZ camera control system for a parliamentary/council chamber. Manages two IP cameras (VISCA protocol) and a BlackMagic ATEM video switcher from a touchscreen operator interface, with a 131-seat auditorium layout for speaker tracking.

---

## Features

- **PTZ camera control** via VISCA over TCP/IP (port 5678) — pan, tilt, zoom, focus, exposure
- **131-seat auditorium layout** with drag-and-drop speaker assignment
- **Camera presets** per seat (1–131) and per speaker (chairman presets)
- **ATEM switcher integration** — monitors program output and triggers automatic camera positioning
- **Session management** — power on/off cameras with motor initialisation sequence
- **Simulation mode** — virtual VISCA servers on loopback for development without hardware
- **Machine-locked password** encryption (PBKDF2-HMAC-SHA256) with login audit log
- **Network camera discovery** via TCP/ARP scan
- **Virtual keyboard** support for touchscreen environments
- **Operating schedule** configuration per weekday

## Configuration

All configuration is stored in plain-text and JSON files in the project root.

### Network / Hardware files

| File | Purpose | Default |
|------|---------|---------|
| `PTZ1IP.txt` | Platform camera IP address | `127.0.0.1` (sim) |
| `PTZ2IP.txt` | Comments camera IP address | `127.0.0.2` (sim) |
| `Cam1ID.txt` | Platform camera VISCA device ID (hex) | `81` |
| `Cam2ID.txt` | Comments camera VISCA device ID (hex) | `82` |
| `ATEMIP.txt` | BlackMagic ATEM switcher IP address | `192.168.1.240` |
| `Contact.txt` | Support contact information | — |

Edit these files with a text editor or use the in-app **Configuration** dialog.

### Data files

| File | Purpose |
|------|---------|
| `seat_names.json` | Speaker names and seat assignments |
| `schedule.json` | Weekly operating hours (Mon–Sun) |
| `chairman_presets.json` | Per-speaker camera presets (name → VISCA preset number) |
| `audit_log.json` | Login attempt history (auto-generated) |
| `password.enc` | Machine-locked encrypted password (auto-generated by `setup_password.py`) |

## Usage

### Launching

```bash
python main.py
```

The application starts with a login screen. The default password is `dublin2024` — change it immediately using `setup_password.py`.

### Simulation mode

Use simulation mode to test the UI and controls without physical cameras.

```bash
python sim_mode.py on     # backs up real IPs, sets loopback (127.0.0.1 / 127.0.0.2)
python sim_mode.py off    # restores real IPs
python sim_mode.py show   # display current IP configuration
```

Simulation can also be toggled from the in-app **Configuration → Simulation** section.

### Camera controls

| Control | Description |
|---------|-------------|
| **SET / CALL mode** | SET selects which camera to control; CALL sends the camera to the selected seat preset |
| **Joystick** | 8-direction digital joystick for pan/tilt |
| **Speed slider** | Adjusts pan/tilt speed (1–18, default 8) |
| **Zoom slider** | Zoom in/out (0–100%) |
| **Home** | Returns camera to default position |
| **Focus** | Toggle Auto / Manual / One-Push AF |
| **Backlight** | Toggle backlight compensation |

### Speaker assignment

Drag a name from the names panel onto a seat in the auditorium layout. Click a seat to move the active camera to that seat's preset position.

### Presets

- Seats 1–131 each map to a VISCA preset number.
- Chairman presets store per-speaker positions: drag a name onto the Chairman seat to recall that person's saved camera position.

## Architecture

### Key modules

| Module | Role |
|--------|------|
| `main.py` | Application entry point — creates QApplication and MainWindow |
| `main_window.py` | Main UI window (1920×1080), composes all panels and controllers |
| `config.py` | Reads config files, defines constants (VISCA port, speed limits, seat layout) |
| `visca_protocol.py` | Pure VISCA command logic — no Qt dependency; fully testable |
| `visca_mixin.py` | Qt adapter layer over ViscaProtocol with UI callbacks |
| `camera_worker.py` | Daemon thread per camera: persistent TCP socket, command queue (max 20), 5 s heartbeat |
| `camera_manager.py` | Centralises camera state (zoom cache, backlight, focus mode, exposure level) |
| `atem_monitor.py` | Background thread monitoring ATEM program output via TCP 9910 |
| `login_screen.py` | Authentication UI with lockout and audit logging |
| `secret_manager.py` | Password encryption (PBKDF2-HMAC-SHA256 + XOR-stream, machine-locked) |
| `hardware_simulator.py` | Fake VISCA TCP servers on loopback — used in simulation mode |
| `sim_mode.py` | CLI and library for toggling simulation mode |
| `setup_password.py` | CLI utility to set or reset the login password |
| `camera_discovery.py` | Network scan (TCP + ARP) to locate camera IPs automatically |

### Threading model

- **CameraWorker** runs a daemon thread per camera. Commands are enqueued from the UI thread and dispatched over a persistent TCP socket. A 5-second heartbeat pings the camera when idle.
- **ATEM monitor** runs a separate daemon thread.
- All UI updates from background threads use `QTimer.singleShot(0, callback)` to safely marshal back to the Qt main thread.

## Security

- Passwords are encrypted with **PBKDF2-HMAC-SHA256** (200,000 iterations) and an **XOR-stream cipher** keyed to the machine's hardware identifier (Windows MachineGuid or hostname). The `password.enc` file is unusable on any other machine.
- The default password is **`dublin2024`**. Change it on first run:
  ```bash
  python setup_password.py
  ```
- All login attempts (successful and failed) are recorded in `audit_log.json` with timestamps.
- To reset a lost password, delete `password.enc`. The application will fall back to the default password on next launch.

## Troubleshooting

| Symptom | Resolution |
|---------|-----------|
| Camera not connecting | Verify the IP in `PTZ1IP.txt` / `PTZ2IP.txt`. Confirm port 5678 is reachable: `nc -zv <camera-ip> 5678` |
| Wrong camera response | Check `Cam1ID.txt` / `Cam2ID.txt` match the VISCA device IDs configured on the cameras |
| ATEM not detected | Verify `ATEMIP.txt` is correct and `PyATEMMax` is installed (`pip install PyATEMMax`) |
| UI does not open | Ensure PyQt5 is installed and `DISPLAY` is set correctly on Linux |
| Login locked | Delete `password.enc` to reset to the default password `dublin2024`, then run `setup_password.py` |
| Simulation mode stuck on | Run `python sim_mode.py off` to restore real IPs |
